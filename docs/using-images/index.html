Responsive Images - Sizes

We can have even greater control over an image by using srcset in conjunction with the sizes HTML5 attribute. By providing a number of versions of an image (at different widths) we can let the browser determine which one to show to the user.

.yaml:

Each entry in sizes must have a size parameter that refers to the actual rendered width of the image at the corresponding media query. In this example, the image's max width will be 516px. Since it's parent's width is, at most, 90%% of the screen width, the image isn't at 516px until the screen is 574px wide. After that, the image will never be bigger than 516px, so the first entry in sizes tells the browser that its size will be 516px at a screen width of 574px and above. The second entry will be used when the browser is less the 574px wide. Since it's the default size, it doesn't need an accompanying media query. Make sure your default size is the last entry in the list.

Notice how, unlike our last example, the srcset entries don't use 1x, 2x, etc. for their descriptor attributes. The descriptor in this example is the width of the image file (using w instead of x). This is required in order for the browsers to determine which version of the image to show based on image width. Order matters when defining your sources. Be sure to list your sources in from largest to smallest.

Note that we're defining the image's width as 100%%, but we told our sizes attribute that it'll be rendered at 90vw (under a screen width of 574px). This is because we want the image to have a width of 100%% of it's parent, but it's parent has a width of 90vw, therefore the image's rendered with will be 90vw as well.

You can test the results by opening up your Network panel in the dev tools and note which version of "image-example-2" is requested by the browser. If viewing on a 1x screen, as soon as the browser renders the image above 516px it will use the larger version of the image since we defined the smaller image as being 516 px in width. If viewing on a 2x monitor, the larger asset will be requested when the image is rendered greater than 258px.

Notice that this example uses a property called aspect_ratio. This is used to predict the space the image will take up once rendered (ratio is height:width). This prevents any content shifts once the image loads. It's not mandatory to include this property, but if omitted, the loaded image will cause the content to jump (if it's within the document flow) - not a great user experience.
